# 1. システム構成・コンポーネント設計
## 推奨技術スタック
### フロントエンド (モバイル): React Native (Expo)
選定理由: iOS/Androidのクロスプラットフォーム開発を単一コードで実現。react-native-reanimatedおよびreact-native-svgによる、HTMLプロトタイプで定義された滑らかなイージング（1.2秒の遷移）やSVGアバターの軽量な描画処理に最適。
### バックエンド / データベース / リアルタイム通信: Supabase (PostgreSQL + Realtime)
選定理由: PostgreSQLベースであり「20年耐える堅牢なスキーマ」を構築可能。標準でWebSocket機能（Supabase Realtime）を備え、インフラ保守の認知負荷と初期コストを極小化できるBaaSアーキテクチャ。
### AIプロキシ (ステートレス処理): Supabase Edge Functions (Deno)
選定理由: 常時起動サーバーを持たず、リクエスト時のみ起動。LLM API実行後、メモリ上の平文テキストを即座に破棄する「ステートレスかつプライバシー・ファースト」な要件を低コストで実現可能。
### 暗号化・E2EE基盤: WebCrypto API / react-native-quick-crypto
選定理由: デバイス側で公開鍵暗号方式（RSA-OAEP / AES-GCM等）を用いたエンドツーエンド暗号化をネイティブパフォーマンスで実行し、運営側から完全なプライバシーを担保する。

## 3. データモデル定義 (DB Schema)
Prisma schema形式によるデータベース定義。「既読(isRead)」カラムを排除し、タイムカプセル用のインデックスを定義する。
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id             String     @id @db.Uuid
  pairId         String?    @db.Uuid
  publicKey      String     @db.Text // E2EE通信用の公開鍵 (PEM形式等)
  createdAt      DateTime   @default(now()) @db.Timestamptz

  pair           Pair?      @relation(fields: [pairId], references: [id])
  journals       Journal[]
  reactions      Reaction[]
}

model Pair {
  id        String    @id @default(dbgenerated("uuid_generate_v4()")) @db.Uuid
  createdAt DateTime  @default(now()) @db.Timestamptz
  users     User[]
  journals  Journal[]
}

model Journal {
  id             String   @id @default(dbgenerated("uuid_generate_v4()")) @db.Uuid
  authorId       String   @db.Uuid
  pairId         String   @db.Uuid
  
  // E2EEにより暗号化された本文（サーバーでの復号不可）
  encryptedNote  String   @db.Text

  // AIパイプラインにより抽出されたメタデータ（ポータブルなJSONB形式）
  metadataJson   Json     @db.JsonB

  // タイムカプセル抽出用の仮想インデックスカラム (フォーマット: "MM-DD")
  monthDay       String   @db.VarChar(5)
  recordedAt     DateTime @db.Timestamptz
  createdAt      DateTime @default(now()) @db.Timestamptz

  author         User     @relation(fields: [authorId], references: [id])
  pair           Pair     @relation(fields: [pairId], references: [id])

  @@index([pairId, recordedAt(sort: Desc)])
  @@index([pairId, monthDay]) // タイムカプセル機能用の複合インデックス
}

model Reaction {
  id        String   @id @default(dbgenerated("uuid_generate_v4()")) @db.Uuid
  authorId  String   @db.Uuid
  journalId String   @db.Uuid
  fxType    String   @db.VarChar(20) // 例: "petal", "light", "breeze"
  createdAt DateTime @default(now()) @db.Timestamptz

  author    User     @relation(fields: [authorId], references: [id])
  journal   Journal  @relation(fields: [journalId], references: [id])
}

### ポータブルJSONスキーマ仕様 (metadataJson)
将来的なデバイス・アプリ移行に耐えうるバージョン管理された構造。

  "context": {

  }

```

## 4. APIインターフェース定義 (REST / Supabase SDK)

### 4.1 感情メタデータ解析 (Edge Function)
平文テキストを送信し、メタデータを受け取る。リクエスト内容はDBに永続化せず、メモリから即座に破棄される。
Endpoint: [POST] /functions/v1/analyze-mood
Request JSON:
```json
{
  "text": "会議が長引いて、夕飯の支度が間に合わなかった。こういう日もある、と呟いてみる。"
}


Response JSON (200 OK):
JSON
{
  "mood": "rough",
  "color": "#DB7093",
  "label": "すこし、こころが揺れているみたい",
  "tone": "soft_warm"
}


### 4.2 ジャーナルの作成 (Supabase PostgREST)
クライアントで暗号化済みのテキストと、AIが生成したメタデータを保存する。
Endpoint: [POST] /rest/v1/Journal (SDK: supabase.from('Journal').insert())
Request JSON:
```json
{
  "authorId": "uuid-user-a",
  "pairId": "uuid-pair-1",
  "encryptedNote": "U2FsdGVkX19+...",
  "metadataJson": {
    "version": "1.0",
    "mood": "rough",
    "color": "#DB7093"
  },
  "recordedAt": "2026-05-01T14:22:00Z",
  "monthDay": "05-01"
}


Response JSON (201 Created):
JSON
{
  "id": "uuid-journal-123",
  "createdAt": "2026-05-01T14:22:05Z"
}
```

### 4.3 リアクションの送信 (Supabase PostgREST)
相手デバイスへのアンビエントなFX（花びら等）通知をトリガーする。
Endpoint: [POST] /rest/v1/Reaction (SDK: supabase.from('Reaction').insert())
Request JSON:
```json
{
  "authorId": "uuid-user-b",
  "journalId": "uuid-journal-123",
  "fxType": "petal"
}
```

## 5. シーケンス図 (処理フローの定義)
ジャーナル入力から暗号化、DB保存、そしてパートナーへのアンビエント通知（WebSocket）までのフローを定義する。
```mermaid

    participant UserA as Client (User A)
    participant Edge as Edge Function (analyze-mood)
    participant LLM as LLM API
    participant DB as Supabase DB
    participant WS as Supabase Realtime
    participant UserB as Client (User B)

    Note over UserA: 入力・マインドフルなUI遷移
    UserA->>Edge: POST /analyze-mood (平文テキスト)
    Edge->>LLM: プロンプト実行

    Edge-->>UserA: JSON返却 (color, mood等)
    Note over Edge: 処理完了。平文はメモリから破棄

    Note over UserA: User Bの公開鍵で平文を暗号化
    UserA->>DB: INSERT Journal (encryptedNote, metadataJson)
    DB-->>UserA: 201 Created

    DB->>WS: Database Insert Event Trigger
    WS->>UserB: WebSocket Event (new Journal)
    
    Note over UserB: プッシュ通知音なし<br/>背景色(metadataJson)のみアンビエント更新
    Note over UserB: ユーザーBがアプリを開いた際に<br/>秘密鍵でencryptedNoteをローカル復号

## 6. 状態管理・データフロー方針
### フロントエンド (React Native)
#### ローカルステート (UI/アニメーション):
react-native-reanimatedのuseSharedValueを使用し、背景グラデーションやSVGアバターのパス変形を管理。JSスレッドをブロックせずネイティブスレッドで描画を実行する。
#### グローバルステート (Zustand):
クライアントの秘密鍵（セキュアストレージからロード）、パートナーの公開鍵、ペアリング状態（pairId）をメモリに保持。
#### データフェッチ・同期 (TanStack Query + Supabase SDK):
ジャーナル一覧の取得状態を管理。取得成功後、インターセプターまたはフック内でZustandの秘密鍵を参照してローカル復号を実行し、平文はReact Queryのキャッシュ内でのみ保持する。
アンビエント通知 (Supabase Realtime):
supabase.channel('public:Reaction').on('postgres_changes', ...)を用いてリアクションを購読。受信時、Zustandを経由せず直接イベントエミッターまたはContextでUIレイヤー（FXコンポーネント）に発火させ、パーティクルアニメーションをトリガーする。

## 7. エラーハンドリング・セキュリティ方針
### エラーレスポンス統一フォーマット
Edge FunctionsおよびSupabaseからのエラーは、以下のインターフェースにラップしてフロントエンドでハンドリングする。
```typescript
interface AppError {
  code: string;       // 例: "E2EE_KEY_MISSING", "NETWORK_TIMEOUT"
  message: string;    // ユーザー向けメッセージ
  timestamp: string;  // ISO8601
  details?: any;      // デバッグ用（本番環境では出力を制限）
}
```

### セキュリティ・プライバシー実装方針
#### エンドツーエンド暗号化 (E2EE):
ユーザーは初回起動時にキーペア（RSA等）を生成。秘密鍵はデバイスのSecure Enclave / Keystoreに保存し、公開鍵のみDBへ登録する。
テキスト送信時、クライアント側で「自身の秘密鍵による署名」と「パートナーの公開鍵による暗号化」を実行。DBには暗号化された文字列のみが渡る。
#### ステートレス同期（既読管理の排除）:
APIおよびDBスキーマには閲覧日時やフラグを一切持たせない。
WebSocketやSilent Pushによる通知は、画面描画（色・アバターの表情変更）のトリガーとしてのみ機能し、システムは「誰がいつ見たか」をトラッキングしない。
#### AIプロキシのメモリ管理:
Deno Edge Functionsでは、リクエスト終了（HTTPレスポンス返却）をもってコンテキストが破棄される特性を利用し、平文テキストのログ出力を意図的に無効化（console.logへのテキスト挿入禁止）して完全なゼロ・ナレッジを担保する。
