# 1. プロダクトビジョンとKPI
## 解決する課題:
忙しさや「照れ」によって生じる夫婦間の感情の断絶、および産後クライシス等の関係悪化。

## コアバリュー:
AIを介した「摩擦ゼロ」の感情視覚化と、非言語的（アバター・エフェクト）なリアクションによる、心理的安全性の高い家族ログ体験。

## 重要目標達成指標 (KPI):
### ペア継続率:
招待したパートナーと共に30日後も利用している割合 50%以上

### リアクション送信数:
1投稿あたりの平均スタンプ/エフェクト送信数 3.0以上

### ネガティブ感情の反転率:
AIジャーナリング後の気分改善スコアの向上

# 2. アクター定義
## パートナーユーザー
夫婦・カップル単位で接続。感情の記録、共有、相互リアクションが可能。

## AIコンパニオン（Tsumu）
ユーザーの感情を解析し、視覚化（色・形）やフィードバックを行うシステムアクター 。

## タイムカプセル・プロセッサー
蓄積されたデータを未来（10-20年後）の資産として結晶化するバックエンド処理 。

# 3. ユーザーストーリーと受入基準 (Acceptance Criteria)
Epic: 摩擦ゼロの感情ジャーナリング
Story: 忙しいユーザーとして、思考を整理し、パートナーに今の状態を伝えるために、最小限の負荷で感情を記録したい。
Acceptance Criteria:
### シナリオ1: 1タップ感情記録（正常系）
GIVEN: 入力画面を開いている。
WHEN: 5段階の顔アイコンから今の気分を選択する 。
THEN: AIが即座にその感情に対応する「色」と「アバターの仕草」を生成し、画面全体に反映する 。
### シナリオ2: 自動メタデータ付加（正常系）
GIVEN: 位置情報とヘルスケア連携が許可されている。
WHEN: 記録を保存する。
THEN: 時刻、場所、天気、その日の歩数が自動でタグ付けされ、入力の手間を省く 。
### シナリオ3: 未入力への非審判的対応（エッジケース）
GIVEN: 3日間記録が途切れている。
WHEN: アプリを起動する。
THEN: 「連続記録（ストリーク）」の途切れを責める演出や「×印」を表示せず、常に「今日から始めよう」という温かいメッセージのみを表示する 。
Epic: 「照れ」を解消する非言語コミュニケーション
Story: パートナーとして、直接言葉にするのは照れくさい感謝や労いを、負担なく相手に届けたい。
Acceptance Criteria:
### シナリオ1: エフェクトによるValidating Response（正常系）
GIVEN: パートナーの投稿（「今日は育児で疲れた」等）が表示されている。
WHEN: 「お疲れ様」スタンプをタップする。
THEN: 相手の画面上に花びらが舞う、あるいは温かい光が包み込むようなフルスクリーンエフェクトを表示する 。
### シナリオ2: アバターによる感情のデフォルメ（正常系）
GIVEN: ユーザーが「怒り」の感情を入力した。
WHEN: パートナーがその記録を閲覧する。
THEN: ユーザー本人の生々しい写真ではなく、少しデフォルメされた可愛いアバターが困った顔をしている状態で表示され、心理的摩擦を和らげる 。
Epic: 感情資産の継承（タイムカプセル機能）
Story: 親として、子供が将来大きくなった時に、自分たちが注いだ愛情を証明するために、記録を資産化したい。
Acceptance Criteria:

### シナリオ1: 過去への接続（On This Day）
GIVEN: 1年以上前の同一日付に記録が存在する。
WHEN: アプリを起動する。
THEN: 「〇年前の今日のふたり」として、当時の感情グラフと写真をノスタルジックな演出（セリフ体フォント等）で提示する 。

# 4. 非機能要件
プライバシーと信頼: * 家族という「聖域」を守るため、エンドツーエンドの暗号化を実装 。
義実家等の「第三者の監視」を完全に排除するため、ペア以外の閲覧履歴（足跡）機能を一切持たない設計 。
ビジュアル・アイデンティティ:
メインカラー: 信頼のブルー（#35%の安堵）と癒やしのグリーンを基調としたアースカラー構成 。
タイポグラフィ: 長期的な記録には「信頼」を感じさせるセリフ体を使用し、操作系にはサンセリフ体を使用する 。
スケーラビリティ:
20年、30年と継続利用されることを前提に、デバイス変更に依存しないマルチプラットフォームでのデータ同期を保証する 。
